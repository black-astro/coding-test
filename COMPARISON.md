## 2. KT_BatchServer가 더 나았던 점 (codeTest에 반영)

1. **실제 백준 문제 번호 매핑** — 학습 신뢰도가 높음. (예: ATM 11399, 토마토 7576, 외판원 2098)
   → codeTest도 문제 메타에 `boj`(문제 번호)·`tier`(세부 티어) 필드를 추가해 채워 넣을 수 있게 함.
2. **세부 티어(B5→B1 …)** — 같은 랭크 안에서도 난이도 곡선이 완만.
   → `tier` 필드 추가(예: "S2"). CLI에서 함께 표기.
3. **정통 코딩테스트 유형의 탄탄한 커버리지**
   - 그리디: ATM, 동전0, 회의실배정, 잃어버린 괄호, 주유소
   - BFS/DFS: 토마토(멀티소스), 벽 부수고 이동(상태 BFS), 적록색약, 숨바꼭질
   - DP: RGB거리, 계단오르기, 포도주, 연속합, 카드구매, 제곱수의 합
   - 백트래킹: N-Queen, 연산자 끼워넣기, 스타트와 링크, 부분수열의 합
   - 그래프: MST(크루스칼), 유니온파인드, 트리의 지름, 케빈 베이컨
   - 고급: 세그트리(합/곱/최대최소/lazy), 비트마스킹 TSP, 중간에서 만나기, 행렬 거듭제곱, 페르마 소정리
   → codeTest는 이미 유사 유형을 보유. **부족하거나 약한 유형은 추가 배치로 보강 가능**(아래 4번).

## 3. codeTest가 더 나은 점 (유지)

1. **3단계 힌트** (1: 접근 방향 / 2: 자료구조·알고리즘 지목 / 3: 거의 정답).
2. **다국어 풀이·채점** (Python/Java/C++).
3. **실측 채점** — 정답 여부뿐 아니라 실행 시간·최대 메모리를 보여주고 제한 초과를 판정.
4. **데이터 주도 구조** — 문제 추가/메타 보강이 쉬움.

## 4. 실제 코딩테스트 조건(시간·메모리) 도입

모든 문제에 다음을 부여(랭크별 기본값, 문제별 조정 가능):

| 랭크 | 시간 제한 | 메모리 제한 |
|------|-----------|-------------|
| Bronze | 1000 ms | 128 MB |
| Silver | 1500 ms | 256 MB |
| Gold | 2000 ms | 256 MB |
| Platinum | 3000 ms | 512 MB |

채점 시 각 테스트케이스마다 **실행 시간(ms)** 과 **최대 메모리(KB)** 를 측정해
- 시간 초과 → `시간 초과(TLE)`
- 메모리 초과 → `메모리 초과(MLE)`
- 출력 불일치 → `오답(WA)`
- 런타임 에러 → `런타임 에러(RE)`
- 컴파일 에러 → `컴파일 에러(CE)`
로 판정하고, 통과 시 worst-case 시간·메모리를 함께 표시한다(실제 백준 결과창과 동일한 감각).

## 5. 다국어 채점 정책

- **표준입출력형(stdin) 문제**: Python · Java · C++ **모두 채점 가능**(가장 실전적인 코딩테스트 형식).
- **함수 구현형(func) 문제**(프로그래머스 스타일): Python으로 채점, Java/C++는 정답 코드 참고용.
- 컴파일러 요구:
  - Java: `javac`/`java` (설치됨, JDK 21)
  - C++: `g++` 필요 — **현재 미설치**. 설치하면 즉시 C++ 채점 활성화(아래 안내).

## 6. C++ 환경 안내 (현재 미설치)

C++ 채점을 켜려면 `g++`(MinGW-w64) 또는 MSVC가 필요합니다. 추천:
- MSYS2 설치 후 `pacman -S mingw-w64-ucrt-x86_64-gcc`
- 또는 winget: `winget install -e --id MSYS2.MSYS2` 후 위 패키지 설치
- 설치 후 `g++ --version` 이 보이면 codeTest가 자동 인식해 C++ 채점을 수행합니다.
